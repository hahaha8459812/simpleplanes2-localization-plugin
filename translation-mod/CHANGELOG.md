# Changelog

## v0.1.4 - 2026-05-01

- 补充控制设置与键位绑定界面翻译，覆盖控制器、键盘、鼠标、视角、移动、飞行控制、武器、设计器工具、镜像、隐藏零件和常用操作绑定。
- 补充标签筛选界面翻译，覆盖 `Aircraft Carrier`、`Civilian`、`Racing`、`Vehicle`、`World War II`、`XML Modded` 等常见标签，并使用上下文规则避免影响其他界面的同名词。
- 调整标签语境术语：`Racing` 译为“竞速”，`Vehicle` 译为“车辆”，保留 `Funky Trees`、`VR` 等专有名词或缩写。
- 为控制器绑定提示增加动态前缀翻译，降低不同控制器输入名称导致的漏翻。
- 分发版默认保持普通翻译模式，不开启文本采集记录，适合普通玩家直接安装使用。

## v0.1.3 - 2026-04-29

- 增加游戏内汉化浮窗面板，支持在游戏内重载翻译、临时开关汉化和打开设置。
- 浮窗面板改为中文显示，并为 IMGUI 面板加载 Windows 中文字体，降低按钮文字显示为方框的概率。
- 支持在面板中启用/关闭快捷键，并自定义重载与开关汉化快捷键，设置会写入 `settings.json`。
- 支持保存浮窗位置；拖动面板后会在松开鼠标时保存，下次启动继续使用上次位置，收起按钮也会跟随该位置。
- 修复关闭汉化后再次开启不立即恢复的问题：开关汉化时会清理场景扫描缓存并主动触发一轮扫描。
- 修复浮窗鼠标拦截方案导致游戏界面被全局拦截的问题，移除透明 Canvas 遮罩，仅保留 IMGUI 面板区域事件消费。
- 补充机身编辑器相关翻译，增加 `Per Corner`，并将 `Rise`、`Run` 调整为“垂直偏移”“水平偏移”。

## v0.1.2 - 2026-04-26

- 扩展零件翻译覆盖：补充螺旋桨发动机、喷气发动机、涡扇发动机、VTOL 发动机、喷气发动机整流罩、主旋翼、直升机尾桨、传动系统、差速器、变速箱、变速器、方向盘、踏板、操纵杆、按钮、开关、握把和座舱盖等相关零件与属性。
- 补充发动机和旋翼相关属性翻译：加力燃烧室、燃烧室温度、涵道比、压缩比、燃油消耗、推力、反推、喷口样式、进气锥样式、桨叶直径、桨叶厚度、桨距控制、油门调速器、尾桨输入等。
- 补充发动机相关悬浮说明翻译，包括涵道比、燃烧室温度、燃油消耗、海平面静推力等说明文本。
- 补充座舱控件相关翻译：控制基座预设、握把控制方案、按钮动作、激活组、武器开火、发射干扰物、起落架、油门和配平等。
- 补充新建作品确认框、保存/加载相关提示和更多设计器页面缺漏文本。
- 优化运行时扫描性能：平时降低场景扫描频率，用户点击、按键、滚轮或切换场景后短时间高频扫描；缓存已处理文本，减少重复翻译和字体处理。
- 修复滑条快速拖动或点击跳转数值后翻译偶尔消失的问题：鼠标松开后延迟 0.1 秒主动扫描一次，并修正翻译后扫描缓存状态。
- 快捷键调整为 `F1` 开关翻译、`F2` 刷新翻译，替换此前容易与游戏或输入法冲突的组合键。
- 重整 README 和开发文档：玩家使用说明前置，开发者与标准翻译流程整合到主 README，Release 包会携带完整自述。
- 继续完善中文字体应用和上下文翻译规则，降低短词误翻以及玩家自定义名称被误翻的风险。

## v0.1.1 - 2026-04-25

- 扩展设计器零件属性翻译，覆盖武器、导弹、火炮、传动系统、灯光、陀螺仪、相机、内饰、仪表、标签和控制器等更多零件。
- 统一若干术语：`Transmission` 译为“变速器”，`Gearbox` 译为“变速箱”，火炮属性页中的 `Cannon` 译为“火炮”。
- 补充飞行界面、干扰物、风力设置、速度显示和多人模式等 UI 翻译。
- 调整多个上下文翻译规则，减少 `Fly`、`Gear`、`Light`、`Basic`、`Open` 等短词在不同页面中的误译。
- 继续完善中文字体应用，保持外部中文字体在翻译文本中的一致显示。

## v0.1.0 - 2026-04-25

- 搭建 `BepInEx + Harmony` 运行时翻译 Mod。
- 支持 `collect`、`translate`、`hybrid` 模式。
- 增加热重载和翻译开关快捷键。
- 增加文本采集上下文：对象路径、父节点路径、组件类型、兄弟序号和锚点位置。
- 内置 `Source Han Sans SC Regular`，解决中文方框和混合字体问题。
- 翻译主菜单、多人入口、服务器浏览器、飞行界面、设计器主要菜单、教程、零件列表、零件属性、悬浮说明、上传发布页和设置页。
- 支持动态文本匹配，用于挡位齿比、发动机偏向、天气、风速、计时和成绩等变化文本。
- 将翻译源拆分到 `zh-CN.fragments`，构建时合并为 `zh-CN.json`。
- 生成 `Dev` 与 `Release` 两种分发包，`Release` 包可直接解压到游戏根目录使用。
